交互总结

Step 用户操作 系统行为
1. 选 Provider 点选卡片 → Next 预填默认 URL + 端口
2. 填 URL 确认或修改 upstream URL → 复制本地地址 → Start 创建 profile + 启动 listener
3. 配客户端 复制 export 命令 → 在终端执行 → 正常使用 agent 自动捕获流量 → 跳转到 workspace
关键设计原则：
① Upstream URL 始终可编辑，预填默认值但不锁死
② 本地地址 + export 命令始终可见可复制
③ 数据流方向（Client → Agent Trace → Upstream）在页面上用图示表达
④ 端口号可以在输入框里直接改（高级用户）
